Deployment guide

Pick how you want to run Dispatch. Every path ends the same way: open https://<host>:8420, set the admin password, add a relay. Copy-paste the commands for your platform.

Not sure which?

  • • Just trying it / already use containers? → Docker.
  • • A Linux or Windows server? → the Linux / Windows installer (sets up SQL for you).
  • • A hypervisor (Hyper-V, VMware, Proxmox)? → import the prebuilt appliance — nothing to install.

🐳 Docker — the fastest start

Brings up Dispatch and its database together. Needs Docker with Compose.

  1. 1. Clone the repo (it has the compose file): 
    git clone https://github.com/chrismuench/Dispatch-SMTP-Relay.git
cd Dispatch-SMTP-Relay
  2. 2. Start everything: 
    docker compose up -d --build
  3. 3. Open https://localhost:8420 (accept the self-signed cert warning). The default login password is in docker-compose.yml — change it after logging in.

Already run an image without the repo? docker run -d -p 8420:8420 -p 8025:8025 -p 2525:2525 -e ConnectionStrings__DispatchLog="Server=…;Database=DispatchLog;User Id=…;Password=…;TrustServerCertificate=True;Encrypt=True" ghcr.io/chrismuench/dispatch-smtp-relay:latest (point it at your own SQL Server / Azure SQL).

🐧 Linux — installer with bundled SQL

A self-contained tarball (no .NET needed). On x64 it can install SQL Server Express for you.

  1. 1. Download & extract the latest Linux tarball from the releases page: 
    tar -xzf dispatch-*-linux.tar.gz
cd dispatch-*-linux
  2. 2a. Install with a bundled SQL Server Express (x64): 
    sudo ./install.sh --install-sql \
  --sa-password 'Strong_SA_Passw0rd!' \
  --admin-password 'Your_Admin_Passw0rd!'
  3. 2b. …or point it at an existing SQL Server / Azure SQL: 
    sudo ./install.sh \
  --sql-connection 'Server=YOURHOST;Database=DispatchLog;User Id=sa;Password=***;TrustServerCertificate=True;Encrypt=True' \
  --admin-password 'Your_Admin_Passw0rd!'
  4. 3. It installs a dispatch systemd service and starts it. Open https://<server-ip>:8420. Logs: journalctl -u dispatch -f.

Passwords must be 12+ chars with upper, lower, and a digit. Omit --admin-password to be prompted (interactive only).

🪟 Windows — single-file installer

One self-contained .exe that installs SQL Server Express, the service, and a firewall rule.

  1. 1. Download DispatchSetup-<version>-x64.exe from the releases page.
  2. 2. Run it (double-click, or silently): 
    DispatchSetup-1.0.0-x64.exe /quiet /log install.log
    CI builds are unsigned — SmartScreen may warn: More info → Run anyway.
  3. 3. Open https://localhost:8420. Logs: C:\ProgramData\Dispatch\logs.

Already have SQL Server? Extract the bundled MSI and point it at your server: DispatchSetup-1.0.0-x64.exe -layout C:\d && msiexec /i C:\d\Dispatch.msi /qn SQLCONN="Server=…;Database=DispatchLog;User Id=…;Password=…;TrustServerCertificate=True;Encrypt=True"

📦 Virtual appliance — import & go

A prebuilt Ubuntu 24.04 + SQL Server + Dispatch VM. Nothing to install — import the image for your hypervisor, power on, and on first boot it configures SQL and starts Dispatch with unique per-VM secrets. Grab the files from the releases page. Recommended: 2 vCPU / 4 GB RAM, a DHCP network. All images are Gen2/UEFI.

Hyper-V — dispatch-appliance-<ver>-x64.vhdx.zip

Unzip the .vhdx, then in PowerShell as Administrator (the helper is in the download):

.\Import-DispatchAppliance.ps1 -VhdxPath .\dispatch-appliance.vhdx -Start

It creates a Gen2 VM, sets the Microsoft UEFI CA Secure Boot template (required for Linux), and starts it. Manual route: New VM → Generation 2 → use the existing VHDX → Security → Secure Boot template = Microsoft UEFI Certificate Authority.

VMware (vSphere / ESXi / Workstation / Fusion) — dispatch-appliance-<ver>-x64.ova

vSphere Client → Deploy OVF Template → select the .ova (or File → Open in Workstation/Fusion). Accept defaults — the descriptor already sets EFI firmware and an Ubuntu 64-bit guest.

CLI with ovftool: ovftool dispatch-appliance.ova vi://user@vcenter/Datacenter/host/esxi.

KVM / libvirt — dispatch-appliance-<ver>-x64.qcow2

Using the bundled helper:

sudo ./import-libvirt.sh dispatch-appliance.qcow2 --start
virsh domifaddr dispatch        # find its IP

Proxmox VE — dispatch-appliance-<ver>-x64.qcow2

Copy the .qcow2 + helper to the Proxmox host, pick an unused VM ID (e.g. 9000):

./import-proxmox.sh dispatch-appliance.qcow2 9000 \
  --storage local-lvm --bridge vmbr0 --start

After it's running — every method

  1. 1. Browse to https://<host>:8420 and accept the self-signed certificate warning.
  2. 2. Set the admin password (first-run screen) — or log in with the one you supplied.
  3. 3. Add a relay (your email provider) and a routing rule in the dashboard.
  4. 4. Point your apps at it — SMTP on 2525 (set 25/587 in the dashboard for production) or the HTTP API on 8025.
